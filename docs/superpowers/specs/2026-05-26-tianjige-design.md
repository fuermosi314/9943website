# 天机阁 — 设计文档

> 工具分类：生活工具 | 工具 ID：`tianjige` | 路径：`/tools/tianjige`

---

## 1. 概述

"天机阁"是一个 3D 场景式物品收纳管理工具。用户通过一个低多边形（Low-poly）风格的 3D 小屋场景，直观地管理家中物品的存放位置。点击家具即可查看/添加物品，支持搜索物品并自动定位到对应家具。

灵感来源：心水收纳 App 的场景式收纳理念，结合 Three.js 3D 渲染打造沉浸式体验。

## 2. 技术方案

- **3D 渲染**：Three.js（通过 `next/dynamic` 动态导入，`ssr: false` 禁用服务端渲染）
- **风格**：低多边形（Low-poly），固定俯视角（等距视角），无实时光影
- **模型**：代码生成简单几何体拼装（BoxGeometry 组合），不依赖外部 GLTF 模型文件
- **材质**：MeshLambertMaterial（有光照感但轻量），无阴影投射
- **数据存储**：IndexedDB（场景、家具布局、物品数据全部本地持久化）
- **框架**：Next.js 14 + React 18 + TypeScript + Tailwind CSS

### 性能控制

- Three.js 通过 `next/dynamic(() => import('./Scene3D'), { ssr: false })` 按需加载
- 场景使用低面数几何体（每个家具 < 500 面）
- 固定视角，不支持自由旋转，减少渲染负担
- 手机端兼容：使用 PointerEvent（而非 MouseEvent）统一鼠标和触屏事件
- 相机移动使用手动 lerp 插值（不引入 GSAP 依赖）
- 遮挡剔除：启用 `renderer.sortObjects = true`

### 边界情况处理

- **空房间**：显示引导文字"点击下方按钮添加家具"，地板正常渲染
- **50+ 物品**：单个家具内使用虚拟滚动（只渲染可视区域）
- **20+ 场景**：场景切换列表可滚动，不溢出屏幕
- **搜索无结果**：显示"未找到匹配的物品"，建议检查其他场景

## 3. 数据模型

```ts
// 场景
interface Scene {
  id: string;           // crypto.randomUUID()
  name: string;         // 场景名称
  emoji: string;        // 场景图标，如 🏠
  isCustom: boolean;    // 是否用户自定义
  sortOrder: number;    // 排序
  thumbnail: string;    // 场景缩略图（自动生成的 3D 截图或默认图标）
  furniture: Furniture[];
}

// 家具
interface Furniture {
  id: string;
  type: FurnitureType;
  name: string;         // 用户可自定义命名
  color: string;        // 自定义颜色（可选，覆盖默认配色）
  position: { x: number; z: number };
  rotation: number;     // 0 | 90 | 180 | 270（只能 90° 旋转）
  scale: number;        // 缩放（0.5 ~ 2.0）
  addedAt: number;      // 添加时间戳
  items: Item[];
}

// 物品
interface Item {
  id: string;
  name: string;
  category: string;     // 物品分类：衣物/书籍/电子/食品/日用/其他
  quantity: number;
  note: string;
  purchaseDate: string; // YYYY-MM-DD
  price: number;
  photos: PhotoRef[];
  createdAt: number;
  updatedAt: number;
}

interface PhotoRef {
  id: string;
  thumbnail: string;    // base64 缩略图（宽 200px）
  width: number;
  height: number;
}

type FurnitureType =
  | 'wardrobe'      // 衣柜
  | 'bookshelf'     // 书架
  | 'shoe-cabinet'  // 鞋柜
  | 'nightstand'    // 床头柜
  | 'drawer-cabinet'// 抽屉柜
  | 'desk'          // 书桌
  | 'dining-table'  // 餐桌
  | 'coffee-table'  // 茶几
  | 'bed'           // 床
  | 'sofa'          // 沙发
  | 'tv-cabinet'    // 电视柜
  | 'fridge'        // 冰箱
  | 'washing-machine'// 洗衣机
  | 'sink';         // 洗手台
```

### IndexedDB 设计

- 数据库名：`tianjige-db`，版本 1
- **scenes 表**：keyPath `id`，存储完整场景数据（含嵌套家具和物品）
- **photos 表**：keyPath `id`，索引 `itemId`，大图 Blob 存储

### JSON 导入/导出

- **导出**：将 scenes 表全部数据序列化为 JSON 文件下载
- **导入**：选择 JSON 文件，校验格式后写入 IndexedDB（合并或覆盖两种模式）
- 导出文件名：`tianjige-backup-YYYY-MM-DD.json`

## 5. 3D 场景设计

### 5.1 房间模型

每个房间由以下几何体拼装：
- **地板**：扁平 BoxGeometry，浅木色，带网格参考线（0.5m 格子，半透明）
- **墙壁**：2-3 面薄墙（后墙+左右墙），白色/浅灰

### 5.2 家具模型

每种家具用 2-4 个 BoxGeometry 组合，配色参考：

| 家具 | 形状描述 | 默认主色 |
|------|----------|----------|
| 衣柜 | 高长方体+两扇门线条 | #8B6914（深木色）|
| 书架 | 多层横板 | #A0522D（棕色）|
| 书桌 | 桌面+四腿+抽屉 | #DEB887（浅木色）|
| 床 | 床架+床垫+枕头 | #E8D5B7（米色）|
| 沙发 | 底座+靠背+扶手 | #4A6FA5（蓝色）|
| 茶几 | 矮桌面+四腿 | #8B7355（木色）|
| 冰箱 | 高长方体+上下门 | #E8E8E8（白色）|
| 鞋柜 | 矮长方体 | #A0522D（棕色）|
| 床头柜 | 小矮柜 | #DEB887（浅木色）|
| 电视柜 | 矮长条柜+电视 | #333333（黑色）|
| 洗衣机 | 方体+圆形门 | #D0D0D0（浅灰）|
| 洗手台 | 台面+盆 | #E0E0E0（白色）|
| 餐桌 | 大桌面+四腿 | #DEB887（浅木色）|
| 抽屉柜 | 多层抽屉 | #A0522D（棕色）|

用户可自定义每件家具的颜色（可选）。

### 5.3 交互

- **点击家具**：Raycaster 检测点击（PointerEvent） → 高亮该家具（MeshBasicMaterial 叠加橙色边缘）→ 弹出物品面板
- **相机**：固定 45° 俯视角，搜索时通过手动 lerp 平滑移动目标位置
- **场景切换**：淡入淡出过渡（opacity 渐变）
- **家具放置**：网格吸附（0.5m 格子），旋转步进 90°

### 5.4 搜索定位

搜索时：
1. 匹配物品名称（模糊匹配）
2. 找到所有匹配物品（跨场景搜索，自动切换到对应场景）
3. 相机平滑移动到该家具位置
4. 该家具边缘发出橙色光芒（#fb6400）
5. 显示面包屑路径：`客厅 > 书桌 > 笔记本电脑`
6. 支持多结果列表，点击切换焦点

## 7. 文件结构

```
app/tools/tianjige/
  └── page.tsx              # 主页面容器（动态导入 Scene3D）

components/tianjige/
  └── Scene3D.tsx            # Three.js 3D 场景 + 所有 UI 面板
                              # （物品面板、物品编辑、场景管理、
                              #   家具选择、家具编辑、搜索、统计）

lib/
  └── tianjige-db.ts         # IndexedDB 封装 + JSON 导入导出
```

**说明**：将所有组件合并到 Scene3D.tsx 中，减少文件数量和组件间状态传递复杂度。Scene3D 内部使用子函数/子组件组织逻辑，但不拆分为独立文件。

## 8. 家具管理

### 添加家具

点击底部"添加家具" → 弹出家具选择面板：
- 按分类展示（收纳类、桌类、其他）
- 每种家具有预览色块和名称
- 点击添加到当前场景中央
- 自动网格吸附到最近 0.5m 格子

### 家具操作

- 点击 → 查看/管理物品
- 长按（500ms）/ 右键 → 编辑（重命名、删除、调整位置/旋转/颜色）

### 家具放置规则

- 位置吸附到 0.5m 网格
- 旋转步进 90°（0° → 90° → 180° → 270° → 0°）
- 缩放范围 0.5 ~ 2.0，步进 0.1
- 家具不可重叠（放置时检测碰撞，重叠时红色提示）

## 9. 场景管理

- 预设场景不可删除，但可重命名、更换 emoji
- 自定义场景可删除（确认弹窗）
- 新增场景：空白房间 + 用户自行添加家具
- 场景排序：长按拖拽调整顺序

## 11. 数据持久化

所有数据存储在 IndexedDB 中：
- 切换场景、添加家具、增删物品均实时写入 IndexedDB
- 页面加载时从 IndexedDB 恢复所有场景和家具数据
- 照片存储方式与 simple-note 一致（缩略图 base64 + 原图 Blob）
- 支持 JSON 导出/导入，防止数据丢失

## 12. 资产统计

统计面板展示：
- **总计**：所有场景的物品总数、总价值
- **按场景分布**：每个场景的物品数量和价值
- **按分类分布**：按 item.category 分组统计

数据从 scenes 表实时聚合计算。
