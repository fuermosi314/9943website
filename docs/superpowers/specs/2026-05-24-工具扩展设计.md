# 9943小工具大全 - 工具扩展设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 扩展网站工具数量，从 3 个增加到 16 个，覆盖图片处理、PDF 处理、生活工具三大类

**Architecture:** 纯前端方案，所有处理在浏览器端完成，无需后端服务器。使用 pdf-lib 处理 PDF，Canvas API 处理图片，自定义状态机实现计算器

**Tech Stack:** Next.js 14, TypeScript, TailwindCSS, pdf-lib, Canvas API

---

## 一、项目背景

当前网站仅有 3 个工具：
- 图片压缩
- 二维码生成
- 字数统计

用户需求：扩展工具数量，覆盖更多场景，供朋友间分享使用

## 二、技术方案

### 2.1 总体架构
- **前端框架**: Next.js 14 App Router
- **样式方案**: TailwindCSS + 自定义 glass-card 样式
- **PDF 处理**: pdf-lib（纯前端）
- **图片处理**: Canvas API + File API
- **计算器**: 自定义状态机

### 2.2 部署方案
- 本地开发：`npm run dev`
- 生产部署：Vercel（零成本，无需服务器）
- 文件处理：全部在浏览器端完成，隐私性好

### 3.1 第一批：生活工具（4个）

| 工具 | 路由 | 功能描述 |

## 六、核心实现细节

### 6.1 专业计算器
```
状态机设计：
- mode: 'standard' | 'scientific' | 'programmer' | 'date'
- display: 当前显示值
- previousValue: 上一个操作数
- operator: 当前运算符
- memory: 存储值

程序员模式额外：
- base: 2 | 8 | 10 | 16（进制）
- 位运算：AND、OR、XOR、NOT、左移、右移
```

### 6.2 单位换算
```
配置化设计：
const units = {
  length: {
    name: '长度',
    units: [
      { id: 'm', name: '米', factor: 1 },
      { id: 'km', name: '千米', factor: 1000 },
      { id: 'cm', name: '厘米', factor: 0.01 },
      // ...
    ]
  },
  // weight, temperature, area, volume, time
}
```

### 6.3 图片处理
```
流程：
1. File API 读取文件
2. Canvas API 绘制图片
3. 执行变换（裁剪/缩放/旋转）
4. canvas.toDataURL() 导出
5. 创建下载链接
```

### 6.4 PDF 处理
```
使用 pdf-lib 库：
- 加载：PDFDocument.load(arrayBuffer)
- 合并：doc1.attachDocument(doc2)
- 拆分：doc.extractPages([0, 1, 2])
- 压缩：保存时启用对象流压缩
```

## 九、约束条件

1. **主界面不动** - 首页 (app/page.tsx)、Header、CategoryNav、ToolCard 等组件暂不修改
2. **只添加新工具** - 新增工具页面 + 更新 lib/tools.ts 注册
3. **保持现有样式** - 新工具使用相同的 glass-card 样式

## 十、注意事项

1. **文件大小限制** - 纯前端处理大文件可能较慢，建议限制 50MB
2. **浏览器兼容** - 使用现代 API，不支持 IE
3. **错误处理** - 文件读取失败、格式不支持等情况
4. **用户体验** - 处理过程中显示加载状态
